# 基于 Python 的机械臂工作空间统一时空占据感知与轻量化风险建模实施计划

## 1. 项目定位

本方案面向机械臂安全运动场景，目标是在**不使用 ROS、不依赖深度学习训练**的前提下，基于 Linux + Python + RGB-D 相机 + 机械臂 SDK，构建一套可运行、可实验、可进一步写论文的原型系统。

系统核心目标是：

\[
RGBD相机 + 机械臂状态
\rightarrow
机器人自身剔除
\rightarrow
外部统一占据建模
\rightarrow
速度估计与短时预测
\rightarrow
轻量化风险表达
\rightarrow
减速/停机或后续局部规划接口
\]

第一版重点不是抓取、不是复杂轨迹优化，而是先解决机械臂工作空间环境感知问题。

---

## 2. 第一版系统边界

### 2.1 第一版暂不做的内容

第一版建议先不做：

- 不使用 ROS；
- 不训练深度学习网络；
- 不做可抓取位姿检测；
- 不做复杂语义分割；
- 不做完整局部重规划；
- 不直接用复杂 mesh 做实时碰撞检测；
- 不追求毫米级感知精度。

### 2.2 第一版要实现的内容

第一版要做到：

**实时感知机械臂工作空间中所有外部占据物体，并输出规划/安全控制可用的简化风险模型。**

---

## 3. 系统输入与输出

### 3.1 系统输入

每一帧输入为：

\[
I_t^{rgb}, I_t^{depth}, q_t
\]

其中：

\[
I_t^{rgb}
\]

为 RGB 图像；

\[
I_t^{depth}
\]

为深度图；

\[
q_t
\]

为机械臂当前关节角。

### 3.2 系统输出

每个外部占据对象输出为：

\[
\mathcal{G}_i(t)=
\{c_i(t), v_i(t), r_i(t), shape_i(t), confidence_i(t), risk_i(t)\}
\]

其中：

\[
c_i(t)
\]

为障碍物中心；

\[
v_i(t)
\]

为障碍物速度；

\[
r_i(t)
\]

为包络半径或安全膨胀半径；

\[
shape_i(t)
\]

为球体、AABB、OBB 或椭球；

\[
confidence_i(t)
\]

为检测置信度；

\[
risk_i(t)
\]

为风险等级。

静态和动态不分开建模。静态障碍只是：

\[
\|v_i(t)\|\approx 0
\]

动态障碍为：

\[
\|v_i(t)\|>v_{th}
\]

## 6. 系统总体流程

主流程如下：

```text
1. 读取 RGB-D 图像
2. 深度图转点云
3. 点云转换到机械臂基坐标系
4. 工作空间裁剪与降采样
5. 根据当前关节角生成机械臂胶囊体模型
6. 剔除机器人自身点云
7. 提取外部占据点云
8. 点云聚类，形成占据对象
9. 对每个对象拟合球体 / OBB / 椭球
10. 时序关联，估计速度
11. 预测未来短时风险占据
12. 计算机械臂与风险占据的最近距离
13. 输出 safe / slow / stop
14. 控制机械臂正常运行、减速或停机
```

# 7. 详细实现步骤

## 阶段 1：相机点云读取与预处理

预计时间：3–5 天

### 7.1 读取 RGB-D 图像

使用 `pyrealsense2` 获取：

```text
color image
depth image
camera intrinsics
depth scale
```

需要保证 depth 和 color 对齐。

输出：

\[
I_t^{rgb}, I_t^{depth}
\]

### 7.2 深度图转点云

根据相机内参：

\[
X=\frac{(u-c_x)Z}{f_x}
\]

\[
Y=\frac{(v-c_y)Z}{f_y}
\]

\[
Z=depth(u,v)
\]

得到相机坐标系点云：

\[
P_t^{cam}
\]

### 7.3 工作空间裁剪

只保留机械臂附近区域，例如：

\[
x\in[-1.2,1.2]
\]

\[
y\in[-1.2,1.2]
\]

\[
z\in[0,1.5]
\]

裁剪目的是减少计算量。

### 7.4 点云降采样

使用体素降采样：

```python
pcd = pcd.voxel_down_sample(voxel_size=0.02)
```

初始参数：

\[
voxel\_size=0.02m
\]

如果后续需要更高精度，可以改为：

\[
voxel\_size=0.01m
\]

### 7.5 阶段验收

完成后应能做到：

- Open3D 实时显示点云；
- 点云数量控制在几千到一两万点；
- 单帧点云处理时间低于 30 ms；
- 帧率达到 10–30 Hz。

---

## 阶段 2：相机到机械臂基坐标系标定

预计时间：1 周

### 7.6 目标

求出：

\[
{}^{base}T_{cam}
\]

使点云从相机坐标系转换到机械臂基坐标系：

\[
P_t^{base}= {}^{base}T_{cam}P_t^{cam}
\]

### 7.7 标定方式

推荐两种方式。

**方式一：标定板固定在机械臂末端。**

机械臂移动到多个姿态，记录末端位姿和相机检测到的标定板位姿，求 eye-to-hand 外参。

**方式二：标定板放置在工作台已知位置。**

用机械臂末端触碰或测量标定板角点，获得其在机械臂基坐标系下的位置，再与相机识别结果匹配。

第一版可以优先使用第二种，简单直观。

### 7.8 验证方法

在 Open3D 中同时显示：

- 机械臂基坐标系；
- 工作台点云；
- 机械臂简化模型；
- 标定板点云。

如果真实工作台和点云位置偏差在 1–3 cm 以内，第一版可接受。

---

## 阶段 3：机械臂简化几何建模

预计时间：1 周

### 7.9 建模目的

外置相机会看到机械臂本体。机械臂运动时，如果不剔除自身点云，就会把自己误判为外部障碍物。

所以要建立：

\[
\mathcal{O}_{robot}(q_t)
\]

### 7.10 建模方式

不要用复杂 mesh。第一版用胶囊体和球串。

每个连杆表示为：

\[
\mathcal{C}_k=(a_k,b_k,r_k)
\]

其中：

\[
a_k,b_k
\]

是连杆两端点；

\[
r_k
\]

是包络半径。

夹爪可以用球体或盒子近似。

### 7.11 配置文件示例

```yaml
robot_capsules:
  - name: base_link
    start: joint1
    end: joint2
    radius: 0.08

  - name: upper_arm
    start: joint2
    end: joint3
    radius: 0.075

  - name: forearm
    start: joint3
    end: joint5
    radius: 0.065

  - name: wrist
    start: joint5
    end: flange
    radius: 0.055

  - name: gripper
    type: box
    size: [0.12, 0.08, 0.10]
```

### 7.12 正运动学实现

输入：

\[
q_t=[q_1,q_2,\dots,q_n]
\]

输出每个关节或连杆关键点：

\[
p_1,p_2,\dots,p_n
\]

第一版有三种实现方式：

1. 用机械臂 SDK 直接读取末端和各关节坐标；
2. 用 URDF + urdfpy；
3. 手写 DH 正运动学。

建议优先用 **URDF 或手写 DH**，不要一开始依赖复杂动力学库。

### 7.13 阶段验收

Open3D 中显示：

- 相机点云；
- 机械臂胶囊体模型；
- 当前关节坐标。

视觉检查机械臂胶囊体是否基本包住真实机械臂。

## 阶段 4：机器人自身点云剔除

预计时间：1–2 周

### 7.14 点到胶囊体距离

对点 \(p\) 和胶囊体 \(\mathcal{C}=(a,b,r)\)，先求点到线段距离：

\[
t=\frac{(p-a)^T(b-a)}{\|b-a\|^2}
\]

\[
t=clip(t,0,1)
\]

\[
p_{proj}=a+t(b-a)
\]

\[
d=\|p-p_{proj}\|-r
\]

### 7.15 剔除规则

如果：

\[
d<\delta_r
\]

则该点属于机器人自身点云，删除。

建议初值：

\[
\delta_r=0.03m
\]

如果标定误差大，可以增大到：

\[
0.04m
\]

### 7.16 输出外部点云

\[
P_t^{external}=P_t^{base}-P_{robot}(q_t)
\]

### 7.17 阶段验收

机械臂运动时：

- 剔除前点云中能看到机械臂；
- 剔除后机械臂本体点云大幅减少；
- 人手、工具、物体不被误删；
- 误删率和漏删率可通过可视化调整。

---

## 阶段 5：统一外部占据对象提取

预计时间：1 周

### 7.18 点云聚类

对：

\[
P_t^{external}
\]

进行聚类。

建议使用 DBSCAN 或 Open3D 的 `cluster_dbscan`：

```python
labels = np.array(
    pcd.cluster_dbscan(eps=0.05, min_points=30)
)
```

初始参数：

\[
eps=0.05m
\]

\[
min\_points=30
\]

### 7.19 聚类后得到占据对象

每个聚类是：

\[
C_i(t)=\{p_1,p_2,\dots,p_N\}
\]

过滤条件：

- 点数太少删除；
- 体积太小删除；
- 距离工作空间边缘太远删除；
- 深度异常删除。

### 7.20 统一表达

每个外部对象统一表达为：

\[
\mathcal{G}_i(t)=\{C_i(t),c_i(t),shape_i(t),confidence_i(t)\}
\]

其中中心：

\[
c_i(t)=\frac{1}{N}\sum_{p\in C_i(t)}p
\]

### 7.21 阶段验收

放入不同物体：

- 人手；
- 工具；
- 箱子；
- 圆柱；
- 泡沫块。

系统都能分割成一个或多个外部占据簇。

## 阶段 6：轻量化几何建模（已完成）

预计时间：1 周 — **实际已实现**

当前实现文件：
- `perception/geometry_fit.py` — `fit_sphere()`, `fit_obb()`, `fit_aabb()`, `create_obb_wireframe()`, `create_sphere_wireframe()`, `create_text_label()`
- `test_clustering_filtering.py` — 单帧和实时模式均集成了 OBB 线框、球体线框、中心标记点显示

可视化效果：
- 每个有效簇显示 **OBB 线框**（与该簇同色）
- 每个有效簇显示 **球形经纬线框**（绿色调）
- 每个有效簇显示 **3D 文字标签**（单帧模式下：簇编号、中心坐标、点数、体积）
- 实时模式下显示 **彩色中心标记点** + OBB/球体线框动态更新

### 7.22 球体模型

球体是第一版安全控制最推荐模型。

\[
\mathcal{S}_i=(c_i,r_i)
\]

\[
r_i=\max_{p\in C_i}\|p-c_i\|+\delta_s
\]

建议：

\[
\delta_s=0.02m
\]

优点：计算最快、最稳定。

缺点：对长条物体比较保守。

### 7.23 AABB 模型

轴对齐包围盒：

\[
\mathcal{B}_{AABB}=(x_{min},x_{max},y_{min},y_{max},z_{min},z_{max})
\]

优点：实现简单。

缺点：物体旋转时不够紧凑。

### 7.24 OBB 模型

用 PCA 求主方向：

\[
R_i=[e_1,e_2,e_3]
\]

将点云投影到主方向坐标系，求长宽高：

\[
l_i,w_i,h_i
\]

得到：

\[
\mathcal{B}_{OBB}=(c_i,R_i,l_i,w_i,h_i)
\]

### 7.25 第一版推荐策略

实际安全判断使用：

**球体模型**

论文对比实验中比较：

- 原始点云；
- 体素点云；
- 球体；
- AABB；
- OBB；
- 球体 + OBB 混合模型。

## 阶段 7：占据对象跟踪与速度估计

预计时间：1–2 周

### 7.26 对象关联

上一帧对象：

\[
\mathcal{G}_j(t-\Delta t)
\]

当前帧对象：

\[
\mathcal{G}_i(t)
\]

用最近邻匹配：

\[
j^*=\arg\min_j\|c_i(t)-c_j(t-\Delta t)\|
\]

如果：

\[
\|c_i(t)-c_{j^*}(t-\Delta t)\|<d_{assoc}
\]

则认为是同一个对象。

建议：

\[
d_{assoc}=0.20m
\]

### 7.27 速度估计

\[
v_i(t)=\frac{c_i(t)-c_i(t-\Delta t)}{\Delta t}
\]

为了降低抖动，用指数平滑：

\[
v_i^{smooth}(t)=\alpha v_i(t)+(1-\alpha)v_i^{smooth}(t-\Delta t)
\]

建议：

\[
\alpha=0.3
\]

### 7.28 统一静动态表达

若：

\[
\|v_i(t)\|<v_{th}
\]

则近似静止。

建议：

\[
v_{th}=0.03m/s
\]

但系统内部不需要单独分静态/动态，仍然统一保存：

\[
\{c_i,v_i,shape_i,confidence_i\}
\]

### 7.29 置信度设计

可以根据以下因素定义置信度：

- 点数越多，置信度越高；
- 连续出现时间越长，置信度越高；
- 速度波动越小，置信度越高；
- 距离相机太远，置信度降低；
- 点云稀疏，置信度降低。

简单形式：

\[
confidence_i = w_1 f(N_i)+w_2 f(age_i)-w_3 f(noise_i)
\]

第一版可以用规则法，不必复杂建模。

## 阶段 8：短时风险占据预测

预计时间：1 周

### 7.30 预测时域

设置：

\[
H=0.5s
\]

预测间隔：

\[
\Delta \tau=0.1s
\]

得到：

\[
\tau_k=k\Delta \tau,\quad k=1,\dots,5
\]

### 7.31 常速度预测

\[
c_i(t+\tau_k)=c_i(t)+v_i(t)\tau_k
\]

静态物体因为：

\[
v_i(t)\approx0
\]

所以预测位置保持不动。

### 7.32 安全膨胀

预测风险半径：

\[
r_i^{risk}(t+\tau_k)=r_i+r_{margin}+\|v_i(t)\|\tau_k+\delta_{unc}
\]

其中：

\[
r_{margin}
\]

是安全距离冗余；

\[
\delta_{unc}
\]

是感知不确定性补偿。

建议初始值：

\[
r_{margin}=0.05m
\]

\[
\delta_{unc}=0.02m
\]

### 7.33 输出风险占据管道

每个对象未来风险表示为：

\[
\mathcal{R}_i(t:t+H)=
\{\mathcal{S}_i(t+\tau_1),\dots,\mathcal{S}_i(t+\tau_K)\}
\]

这也是论文里的一个重要亮点：

**不只给当前障碍物，而是给未来短时风险占据。**

---

## 阶段 9：机械臂距离评估与安全策略

预计时间：1 周

### 7.34 当前最近距离

机械臂连杆胶囊体：

\[
\mathcal{C}_k(q_t)
\]

障碍物风险球体：

\[
\mathcal{S}_i(t)
\]

计算：

\[
d_{min}(t)=\min_{k,i}d(\mathcal{C}_k(q_t),\mathcal{S}_i(t))
\]

### 7.35 未来风险距离

如果有机械臂参考轨迹：

\[
q_{ref}(t+\tau)
\]

则计算：

\[
d_{risk}=
\min_{\tau_k,k,i}
d(\mathcal{C}_k(q_{ref}(t+\tau_k)),\mathcal{S}_i(t+\tau_k))
\]

第一版如果没有未来轨迹，就先用当前距离：

\[
d_{risk}=d_{min}
\]

### 7.36 风险等级

设置：

\[
d_{safe}=0.15m
\]

\[
d_{slow}=0.10m
\]

\[
d_{stop}=0.05m
\]

规则：

\[
d_{risk}>d_{safe} \Rightarrow SAFE
\]

\[
d_{slow}<d_{risk}\leq d_{safe} \Rightarrow WARNING
\]

\[
d_{stop}<d_{risk}\leq d_{slow} \Rightarrow SLOW
\]

\[
d_{risk}\leq d_{stop} \Rightarrow STOP
\]

## 阶段 10：机械臂运动控制接口

预计时间：1–2 周

### 7.37 读取机械臂状态

通过机械臂 Python SDK 或 TCP 接口读取：

\[
q_t,\dot q_t
\]

如果暂时无法实时读取，可以先手动设置机械臂姿态或使用离线记录数据验证感知系统。

### 7.38 生成简单点到点轨迹

第一版不接复杂规划器，自己生成点到点轨迹。

使用五次多项式或 minimum-jerk：

\[
q_{ref}(t)
\]

\[
\dot q_{ref}(t)
\]

\[
\ddot q_{ref}(t)
\]

### 7.39 速度缩放

根据风险距离生成速度缩放因子：

\[
s(t)=clip\left(
\frac{d_{risk}-d_{stop}}
{d_{safe}-d_{stop}},
0,1
\right)
\]

然后：

\[
\dot q_{cmd}=s(t)\dot q_{ref}
\]

如果 SDK 支持速度控制，直接发送：

\[
\dot q_{cmd}
\]

如果只支持位置控制，则小步更新：

\[
q_{cmd}(t+\Delta t)=q_{cmd}(t)+\dot q_{cmd}\Delta t
\]

### 7.40 停机策略

当：

\[
d_{risk}\leq d_{stop}
\]

执行：

- 停止轨迹推进；
- 发送 stop/pause 命令；
- 机械臂保持当前位置；
- 风险解除后继续执行。

# 8. 线程设计

为了提高实时性，建议分成三个线程。

## 8.1 线程 1：相机与点云线程

频率目标：10–30 Hz

负责：

- 读取 RGB-D；
- 生成点云；
- 坐标变换；
- 裁剪；
- 降采样。

## 8.2 线程 2：占据建模线程

频率目标：10–20 Hz

负责：

- 机器人自身剔除；
- 外部点云聚类；
- 几何拟合；
- 对象跟踪；
- 速度估计；
- 风险预测。

## 8.3 线程 3：安全控制线程

频率目标：50–100 Hz

负责：

- 读取当前关节角；
- 更新机械臂胶囊体；
- 计算距离；
- 输出 safe / slow / stop；
- 控制速度缩放或停机。

这样即使相机只有 15 Hz，安全线程也可以用最近一次障碍物模型持续计算风险。

# 15. 方案总结

本方案的核心价值是：

1. **不依赖 ROS 和深度学习训练**，适合当前算力和设备条件；
2. **利用机械臂自模型剔除自身点云**，解决外置相机下机械臂自身误检问题；
3. **统一静动态占据表达**，把静态障碍看作速度接近 0 的占据对象；
4. **将原始点云转换为轻量化几何模型**，便于实时距离计算和后续规划；
5. **引入短时风险占据预测**，为减速、停机和后续局部重规划提供输入；
6. **可自然扩展到 ROS、MoveIt、局部轨迹优化、动态抓取和六维力柔顺执行**。
